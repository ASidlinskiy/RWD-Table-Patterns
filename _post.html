Intro: Tables that display complex data can play nicely with a responsive design, with a little help. Using progressive enhancement and responsive design methods, we developed a way to render a complex table that displays comfortably on small screens, provides quick access to all data points, and preserves the visual relationships between the headings and columns of data that we (or the user) deem important. On smartphones, a complex HTML data table displayed at its full width may require zooming or scrolling, which in a way defeats the purpose of displaying data in a table. Visual relationships between headings and cells, and between neighboring columns, are often crucial to understanding the data. When whole sections of the table are out of view the experience can be frustrating. Chris Coyier developed a responsive CSS workaround that leverages media queries and CSS-generated content to display table cells and their headers in a vertical list on smaller screens. Table data is infinitely more readable on a mobile device when rendered this way, however, in many cases, the relationships among columns of data are just as important as the connection between headings and cells. So we decided to find a way to keep the basic table structure in place -- with headings above whole columns that sit side-by-side -- at all screen sizes. Solution: It's easier to explain with a concrete example, so we'll use a table that lists technology companies and their stock pricing. Each row displays data for a single company; columns organize data points by type. Our goal is to show as many columns of data as will fit on the screen. The entire table will fit on larger screens -- no problem there. But smaller tablet and smartphone screens can fit only a subset of columns comfortably, in which case we'll need to hide columns of less essential data and provide a way to access them easily. To do that we'll create a menu for selecting which columns to show. Before we dive into code, first we should define which data is essential (always visible) or optional (shown only if space allows). In our table of tech companies, the data is somewhat meaningless without the company name or current stock value.
